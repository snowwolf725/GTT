JOE Release Notes

1.8.10.6 - 6/03/04

Bug Fixes:
  1. Fixed a bug where I was not assuming object reuse by XML parsers which caused node attribute to not get parsed correctly and thus resulted in mangled data when opening an OPML and OML document.
  2. Fixed bug with file revert where it was resizing and re-positioning the document when it should not have been.
Feature Enhancements:
  1. The merge menu items now enable and disable themselves when they may be used.

1.8.10.5 - 5/18/04

Bug Fixes:
  1. Fixed a bug in the way character data is accessed in the XMLProcessor base class which was causing document attributes to not get saved correctly in OPML and OML files when an & character appeared in them. This should also fix situations where multiple attributes where being created when they should not have been on opening an OPML or OML file.

1.8.10.4 - 5/17/04

Bug Fixes:
  1. Fixed a bug in Find/Replace where doing a "start from top" where the length of the last line in the document was less than the current cursor position within the current node would cause an out of bounds exception.
  2. Fixed a bug in Find/Replace where the use of the '/' character in a regular expression was being escaped where it shouldn't have been causing character patterns such as /s, /w, etc. to not work correctly.
  3. Fixed a bug where the backspace was not being consumed correctly in german locales, and possible other non-english locales. The patch was provided by Hans-Ulrich Teufel.
  4. Fixed various text chunks that were not setup for internationalization.
  5. Fixed a minor bug where the "Save As" file chooser was not being set to the most recent save directory for untitled documents.
  6. Fixed a bug where the "Merge with Spaces" action was not applying the spaces to descendant nodes.
New Features:
  1. Documents that were open when you last closed JOE will reopen themselves automatically the next time you open JOE. This behaviour can be enabled/disabled via a new preference called "Reopen documents on startup" on the "misc" preferences panel.
Feature Enhancements:
  1. Added tabs to the Find/Replace frame for the three search scope. Generally cleaned up the layout of this screen and it's associated dialogs.
  2. Changed "Merge with Spaces" to "Merge with Delimiter" and added preferences to the "Misc" panel for setting the delimiter, enabling/disabling trimming of whitespace, and enabling/disabling the inclusion of empty nodes.
Other Changes:
  1. We've got a more extensive and correct german translation file thanks to Hans-Ulrich Teufel.
  2. We've got a more extensive and correct japanese translation file thanks to ka2hiro.

1.8.10.3 - 02/23/04

Bug Fixes:
  1. Fixed the line endings in run_joe.sh so that they are unix rather than windows.
  2. Fixed a bug where the menu item used to toggle attribute display was not staying in sync with the state of the current document as a user changed from one document to another.
  3. In the process of reworking the recent files list I believe I fixed the problems with window maximization and preservation of size and position both when opening from the recent files list and OPML.
Feature Enhancements:
  1. Rewrote the interface for spell checking. Added support for adding words to the dictionary. Added a "cancel" button. Changed from a combobox to a textfield and list model for selecting replacements. Added tooltips for buttons. Fixed minor bugs as well.
  2. Incorporated a patch provided by Axel Rauschmayer which makes the accelerator keys more platform independent, and which enables the top-level menubar in Mac OS X.
Other Changes:
  1. Updated BeanShell to version 1.3.0 from version 1.2b7.
  2. Updated jakarta-oro (Perl regular expression support) to version 2.0.8 from version 2.0.6.
  3. Updated Jazzy spell checker to version 0.5 from version 0.4.
  4. Reworked parts of the package structure which were annoying me and added more extensive java docs.
  5. Reworked the recent files list to use an XML config file rather than a serialized java file. This should be the last release that causes recent files lists to become invalid after an upgrade.
  6. Updated the default configuration to use UTF-8 as an encoding type and turned off some of the less commonly used features. Basically I setup the default configuration to be more like the way I configure JOE myself.
  7. Finally added a release target for ANT and some release instructions, so doing releases should be a little less time consuming.

1.8.9.2 - 09/01/02

New Features:
  1. Added text actions for moving the current selection. Ctrl-Right Arrow will move the current selection to the right one character, or down to the next line if the selection is at the end of the current text area. Left Arrow works similarly.
  2. Added text actions for expanding a selection within a text area with the keyboard. Shift-Up Arrow Selects to the start of the text area. Shift-Down Arrow selects to the end of the text area.
  3. Added text actions for expanding a selection within a text area with the keyboard. Right Arrow expands one character to the right. Ctrl-Shift-Right Arrow expands to the next work boundary. Left Arrow works similarly. This functionality existed prior to the addition of the Action code so this is sort of a bug fix as well.
Feature Enhancements:
  1. Undo/Redo now displays informative text in the MenuItems about what you are undoing and redoing.
  2. Undo All/Redo All now displays your current location within the undo queue, as well as the maximum size of the undo queue.
  3. Resizing the max undo queue size no longer clears out the redoables unless unless it proves necessary.
Bug Fixes:
  1. Fixed a bug with JDK1.4 where focus wouldn't change to the hidden component, causing the cursor to track with you as you scroll, rather than staying with the node you were editing.
  2. Fixed a bug introduced in 1.8.9.1 where the default text action wasn't working in JTextAreas. This made it impossible to enter text in various places, such as find/replace and some of the preferences panels.
  3. Fixed some StackOverflow issues with demoting thousands of nodes at the same time, by unwinding more of the recursive methods in NodeImpl.
  4. Fixed an obscure bug with CompoundUndoableReplace where deleting the first node in the tree when it is expanded could cause an exception when you did a "redo" where the previous redo selected a node.
  5. Fixed a bug where adjusting the max size of the undo cache to a smaller number than the current undo cache size would leave an extra redo hanging around thus causing an exception if you redid it.
  6. Fixed a bug where redoing a Node Attribute modification would cause it to become read only.
  7. XML-RPC Macros now remove the last line ending for the {$value} replacement when a node is selected. Fix for bug #601682.

1.8.8 - 04/18/02

New Features:
  1. Automatic node level created/modified dates. If turned on via application or document level preferences, node attributes for creation and modification dates will be generated. By default this feature is turned off.
  2. Read-only property for node and document attributes. Note: node level settings take precedence, so if the node itself is read-only, you won't be able to adjust attribute level settings.
  3. New Selection key commands. You can now use the keyboard to make selections. It is possible to create disconnected selections from the keyboard by using the left and right arrow commands described below.

    The following is a list of the key commands added.
    1. shift-up: extends the selection up.
    2. shift-down: extends the selection down.
    3. ctrl-shift-up: reduces the selection from below.
    4. ctrl-shift-down: reduces the selection from above.
    5. shift-left: deselects the current youngest selected node and extends the selection up.
    6. shift-right: deselects the current oldest selected node and extends the selection down.
    7. ctrl-shift-left: deselects the current oldest selected node and reduces the selection from below.
    8. ctrl-shift-right: deselects the current youngest selected node and reduces the selection from above.
    To make room for the shift commands, the move commands from previous versions had to be re-mapped. The following is a list of the new move keyboard commands:
    1. ctrl-up: Moves current selection up but stays at the current depth.
    2. ctrl-down: Moves current selection down but stays at the current depth.
    3. ctrl-left: Moves current selection up, adjusts depth as needed.
    4. ctrl-right: Moves current selection down, adjusts depth as needed.
  4. Find/Replace has had a major rewrite. It now support searching over the current document, all open documents, or the file system. The file system find/replace supports file and directory level glob filtering for both inclusion and exclusion (Now I won't mess up my CVS entries when doing a replace). Results are also displayed in a results dialog that lets you jump to the matched text in any of the open document. Currently the filesystem results are displayed, but don't automatically open the file from the file system. The file system find/replace functionality also makes use of a progress dialog that allows you to cancel a find/replace in progress. One caveat, large find/replace operation may cause out of memory errors. Recovery from this is graceful since you can just close the progress dialog and the current results will be displayed. By large I mean 50,000+ matches on searches over 10,000+ files. From what I've seen, the size of the result set is what causes the out of memory error due to the large number of objects created.
  5. Added more new key commands.
    1. shift-return: insert new node above.
    2. shift-ctrl-return: Split current node at cursor and insert above.
  6. New BSH Macro, "Format Markup". This macro will reformat XML/HTML markup into a nice clean hierarchy of nodes. Still a bit rough, but worth a try if you've got markup to format.
Feature Enhancements:
  1. Script and Macro editor now have save commands so you don't have to close the script editor to save the script.
  2. Inserting a new node when nodes are selected now always puts the new node as the next older sibling of the oldest selected node. Inserting a new node when in text selection mode and the current node is an open branch node still inserts the new node as the first child of the current node rather than as an older sibling.
  3. Pasting now works the way inserting new nodes do. See #2 above for more detail.
  4. File Export/Save has been modified so that File Formats can throw up there own dialog after the user has gone through the file chooser stage. By returning a null for bytes[] the save/export can be aborted, so user's may cancel at this point if the FileFormat code is written to support it. The updated HTMLExportFileFormat is an example of this.
  5. The Export HTML file format has been rewritten to support document templates stored on disk. Now the user is prompted to select a style to use when they export. The templates are stored in prefs/css/.
  6. The "Render Image Manifests" script now outputs image aspect ratios and will provided width x height for each image normalized to a fixed area.
Bug Fixes:
  1. Fixed a bug where cancelling an "Export" would cause the next save to fail. Same bug also fixed in "Export Selection" where cancelling would also cause you to lose the document tree except for what was selected.
  2. Fixed bug where text selection would be lost whenever the user moused over the "Recent Files" menu.
  3. Fixed bug where the "inconsistent hierarchy" message was blank.
  4. Fixed bug where File Exporters weren't getting all the "meta" data they should have from the DocumentInfo object. This was the result of creating a new DocumentInfo Object rather than cloning the existing one.
  5. Fixed bug where the "export selection" menu item was not being disabled after a save when there was no selection.
  6. The "Hoist" menu item now gets enabled only when you can actually use it.
  7. Fixed a bug where doing a delete that merges into the previous line would cause cursor focus to be lost on a subsequent scroll action. This was an error in UndoableEdit so this may have fixed problems elsewhere as well.
  8. Pasting into other apps now will not have extra indenting.
  9. Fixed bug #527137 where help docs wouldn't load on linux. Thanks to sjakob for the fix.
  10. Fixed bug where document level attributes with an empty value would not get loaded for an OPML document.
  11. Fixed some bugs in the way application and document level settings were working.
  12. Updated run_joe.sh, provided by Gary Benson (dirtyrat), so that it now handles paths more sensibly, so you can invoke it as: run_joe ../../file.opml.

1.8.7 - 02/04/02

License Change:
  1. Switched to a BSD license.
  2. We are now building our own jmousewheel.jar library. Code separation is much cleaner so that there is now only a "uses" relationship between outliner.jar and jmousewheel.jar. This should make things compatible between the use of a BSD license for outliner.jar and the LGPL for jmousewheel.jar.
New Features:
  1. Support for multiple File Protocols added.
    1. Implemented LocalFileSystemFileProtocol for opening/saving/exporting to the local filesystem.
    2. Implemented WebFilePHPFileProtocol for opening/saving/exporting to Steven Spencer's PHP based remote file storage system. The WebFile code is now contained in it's own jar: com.yearahead.io.webfile.jar.
  2. Added display options for document window titles. User can now set outline document window titles to show a file's full pathname, a truncated version of the full pathname, or just the file name. Setting is done via a Document Titles widget at the bottom of the Application Preferences Look & Feel panel.
  3. Added file import functionality
    1. Added new File menu submenu, Import. Import opens up a format that JOE can read but not write.
    2. Added import format Brain Forest Palm pdb. Import reads node and note text, ignores other data.
    3. Added import format Shadow Plan Palm pdb. Import reads node and note text, ignores other data.
    4. Added import format Thought Manager Palm pdb. Import reads node and note text, ignores other data.
  4. Added Maximize:Horizontal/Vertical commands to Windows menu. These commands resize the topmost window so that it is made as large as fits within JOE along the specified axis. The window size does not change along the other axis.
  5. Added Tile Horizontal/Vertical/Grid commands to Windows menu. Each command tiles all non-iconified document windows. The axis tilings maximize the windows along the specified axis. The grid tiling seeks a balanced arrangement.
Enhanced Features:
  1. Autosave agent frequency can now be set from the misc panel.
  2. Added a "ding" when a search/replace finds no matches.
  3. Andreas Hochsteger (highstick) has provided a German translation of the gui_tree file so that JOE can be run with a German Text in the interface.
  4. Enhancements to Recent Files list user interface. Users can now set four user-interface properties: list size, ordering, direction, and name form. Setting is done via a Recent Files panel within the Application Preferences Open & Save folder.
  5. Enhancements to Application Preferences Open & Save panel. New widgets let users specify Default Encoding and Default Format separately for Open, Import, Save, and Export operations.
  6. Added sub-containers to preference panels so that prefs can be grouped more intelligently.
Bug Fixes:
  1. Fixed a bug where new documents would often have the text area hidden until the next redraw.
  2. Fixed a bug where open Help files killed the Autosave Agent script.
  3. Fixed a null pointer bug in the class OutlinerCellRendererImpl.
  4. Fixed a null pointer bug in the method StringTools.replace()
  5. Fixed several out-of-range string index bugs in the class TextKeyListener
Notes:
  1. If you have a previous version of JOE on your system you will probably need to manually copy the updated autosave agent script from "/prefs/scripts/Autosave Agent.txt" to "/outliner/scripts/Autosave Agent.txt". If you don't do this it just means the autosave agent won't work.
  2. The sources for com.organic.maynard.jar have been put in CVS. Xerces.jar is required to compile com.organic.maynard.jar, but I'm leaving it out of the builds because it is very large (1.7M) and is not used by JOE. I will check in the license and a copy the copy of xerces.jar I'm currently using, so you can download it from CVS or instead get it directly from Apache: http://xml.apache.org/xerces-j/index.html. I'm currently using xerces 1.4.3, but any recent release should be fine.
  3. We've updated the way the recent files list works, so your current recent files will get blown away if you use the new version.
  4. We're now using Ant to build.

1.8.4 - 09/04/01

Bug Fixes:
  1. Fixed a bug where changing the name of a macro during an "update macro" operation would not delete the old macro from the file system.
  2. When you had a set of nodes selected and you typed a few characters quickly they would not all get sent to the youngest node in the selection. This was a result of the youngest node not being set as the current editing node.
  3. Added alpha-numeric validation to attribute key in the new attribute dialog. If a non XML compliant key was used it would make the OPML unparseable and thus unreadable.
  4. Fixed a bug where doing a find/replace with start at top selected would not search over the descendants of the last child of root.
  5. The installation code now copies macros files one by one. If an existing macros.txt file exists then lines for any new macros are now appended. Prior to this, people who were installing over old builds wouldn't get any new macros that were added unless they copied them manually.
  6. Added validation for macro names so that a user can't create a name that would be an invalid filename. Prior to this you could get an exception and then your macros.txt file would be out of sync with the filesystem.
New Features:
  1. Added sorting macros. These show up in the macro popup under "sort". These are implemented in BSH so you can easily roll your own.
  2. Added document attributes. Now you can store arbitrary document level data via attributes. These will be saved/loaded when you use OPML as your file format.
  3. Added regular expression support to Find/Replace via Apache's jakarta-oro package.
  4. You can now save Find/Replace items.
Feature Enhancements:
  1. The XML-RPC Macro now uses a single URL rather than server and port. This will allow for more flexibility in connecting to a server. However, this breaks any existing XML-RPC Macros. You will need to update any existing macros with the appropriate URL before using them.
  2. More enhancements to the XML-RPC macro. Now you can put the body of the XML-RPC call within the macro. The current selection can be passed in and will substitute itself for {$value}. Inline replacement is improved. Return values now convey the correct hierarchy of and .
  3. Finished pulling out all the text from the GUI into the gui_tree.xml file. Now it is possible to do the multiple language thing in a more comprehensive manner.
  4. Made the macro popup menu only work if you click on a selected node or the current node you're editing. This should make it's behaviour a little more intuitive.

The windows include: MacroManager, Application Preferences, Document Preferences, Find/Replace, MacroEditor, Document Statistics. Also, these all have a common base class so behaviour can be managed more consistently.
  5. Added a read-only node option to Find/Replace and in the process reconfigured the GUI layout. Also fixed the focus when you find a match.
  6. Rewrote the "goto" dialog. It now supports column numbers as well as line numbers.
